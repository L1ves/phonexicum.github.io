# Threat Intelligence (rss/podcasts/news)

***Threat intelligence*** - evidence-based knowledge, including context, mechanisms, indicators, implications and actionable advice, about an existing or emerging menace or hazard to assets that can be used to inform decisions regarding the subject's response to that menace or hazard.

[Cyber Threat Intelligence](http://www.forensicswiki.org/wiki/Cyber_Threat_Intelligence)

* ***IOC - Indication of Compromise***

    * Exists several systems for storing and exchange IOCs

        * [Open IOC db](https://openiocdb.com/)
        * [STIX - Structured Threat Information Expression; TAXII - Trusted Automated Exchange of Intelligence Information](https://oasis-open.github.io/cti-documentation/)
        * [CAPEC - Common Attack Pattern Enumeration and Classification](https://capec.mitre.org/about/index.html)
        * IODEF - Incident Object Description Exchange Format

    * [Yara](https://yara.readthedocs.io/) - tool aimed at helping malware researchers to identify and classify malware samples
        <br> defacto looks like standard for storing IOC, many other tools supports it
    * ***CIF (Collective Intelligence Framework)*** - a cyber threat intelligence management system which allows you to combine known malicious threat information from many sources and use that information for identification (incident response), detection (IDS) and mitigation (null route)
    * [Sigma](https://github.com/Neo23x0/sigma) - Generic Signature Format for SIEM Systems (*"Sigma is for log files what Snort is for network traffic and YARA is for files"*)

* RSS feeds

    * [SANS diary](https://isc.sans.edu/xml.html) - vulnerability rss feed
        <br> [SANS stormcast podcast](https://isc.sans.edu/podcast.html) - daily 5-10 minute information security threat updates
